# Reference: Design Mapping

设计稿映射不是“截图翻译成 JSX/CSS”，而是把视觉目标映射成当前项目中的页面结构、组件结构和业务状态。

## 1. 先找对应关系，再谈实现

需要先回答：

- 设计稿对应哪个页面、哪个路由、哪个弹窗或哪个局部模块？
- 设计稿中的每个主要区域由谁承载：页面本身、父级 layout、公共组件，还是本次新增组件？
- 设计稿是否展示了完整壳层，但项目里这些壳层其实已经统一实现？

## 2. 识别哪些内容不该动

常见不应改动区域：

- 顶部 header
- 侧边栏
- 全局 breadcrumb
- 公共 page shell
- 已被多个页面复用的搜索、筛选、表格或表单壳层

用户没有明确要求时，默认只改目标内容区。

## 3. 缺失设计如何推断

允许推断的常见部分：

- 空态
- 加载态
- 错误态
- 禁用态
- hover / active / selected 状态
- 响应式收缩规则
- 局部组件的次级排列

推断规则：

- 优先参考同一设计稿中的相邻模块
- 其次参考同一项目现有页面和组件
- 再参考现有主题变量、组件尺寸和交互规范
- 不要凭个人审美自由发挥新的视觉语言

## 4. 统一主题风格

检查以下要素是否能映射到项目现有体系：

- 颜色、渐变、边框、阴影
- 字号、字重、行高、字距
- 圆角、间距、栅格、卡片层级
- 按钮、输入框、标签、列表、表格、弹窗
- icon 风格、装饰图形、背景纹理

如果设计稿风格与项目现有风格存在差异，要先判断：

- 这是本次需求要引入的新主题，还是设计稿作者的局部探索？
- 项目是否允许局部页面偏离全站主题？
- 如无明确说明，默认与现有项目风格对齐，而不是把整页做成另一套设计语言。

## 5. 差异映射结果至少要包含

- 页面/区域级对应关系
- 组件级对应关系
- 缺失设计的推断项
- 需要新增的组件
- 可以复用的现有组件
- 不应触碰的公共壳层
