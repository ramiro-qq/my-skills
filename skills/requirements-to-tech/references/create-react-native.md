# React Native 项目创建参考

## 使用场景

当需求涉及从零创建 React Native 项目时，先读取本参考，再把建项方案写入技术方案文档。

## 原则

- 必须先完成技术方案，再进入建项阶段。
- 先判断是纯移动端应用、现有系统移动端补充，还是跨端共享项目。
- 先确认导航、状态管理、网络请求、UI 组件体系和平台差异处理方式。

## 技术方案中至少要说明

- 目标平台：iOS / Android / 双端
- 建项方式与基础栈
- 路由 / 导航方案
- 请求层与异步状态方案
- 组件体系与主题方案
- 与原有 Web / 后端系统的接口关系
- 目录结构建议
- 初始化步骤建议
- skills 安装占位

## 输出要求

如果最终推荐创建 React Native 项目，技术方案中需要明确平台差异、构建要求、调试要求和后续验证方式。
