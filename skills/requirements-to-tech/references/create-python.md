# Python 项目创建参考

## 使用场景

当需求涉及从零创建 Python 项目时，先读取本参考，再把建项方案写入技术方案文档。

## 原则

- 必须先完成技术方案，再进入项目初始化。
- 先判断项目类型：脚本工具、Web 服务、任务调度、数据处理、AI 服务或 SDK。
- 技术栈不要先入为主，优先根据用户需求和项目约束决定。

## 技术方案中至少要说明

- 项目类型与目标
- 运行方式与部署方式
- 目录结构建议
- 依赖管理方式
- Web 框架或任务框架选择理由
- 配置管理、日志、测试、代码规范方案
- 初始化步骤建议
- 后续需要安装的 skills 占位

## 输出要求

如果最终推荐创建 Python 项目，技术方案中要单列“项目初始化方案”章节，避免只写业务功能，不写工程落地方式。
