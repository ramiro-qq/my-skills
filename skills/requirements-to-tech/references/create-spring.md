# Java 项目创建参考

## 使用场景

当需求涉及从零创建 Java 项目时，先读取本参考，再把建项方案写入技术方案文档。

## 原则

- 必须先完成技术方案，再进入建项阶段。
- 优先先判断项目类型：单体服务、后台管理服务、网关、任务服务、API 服务或集成服务。
- 先明确接口边界、数据存储、配置管理、日志、监控和部署方式。

## 技术方案中至少要说明

- 项目类型与职责边界
- 推荐框架与原因
- 构建工具与依赖管理方式
- 分层结构与目录结构建议
- 配置、日志、监控、测试方案
- 对外接口与错误处理约束
- 初始化步骤建议
- 后续 skills 安装占位

## 输出要求

如果最终推荐创建 Java 项目，技术方案中必须体现工程结构、环境配置和部署约束，而不是只写业务逻辑。
